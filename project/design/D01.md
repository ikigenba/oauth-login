# Decision 1 â€” Package decomposition and seams

## Decision.

Four packages. Each owns one failure mode and is exercisable without the
others.

    cmd/oauth-login     flags, validation, composition root, streams, exit codes
    internal/oauth      PKCE, authorize URL, token exchange
    internal/callback   loopback listener, redirect capture
    internal/browser    launch seam

`internal/oauth` and `internal/callback` are independent: neither imports the
other. `cmd/oauth-login` is the only package that imports all three, the only
one that touches `os.Stdout`/`os.Stderr`/`os.Exit`, and the only one that
constructs real defaults (`crypto/rand.Reader`, the real launcher, the real
deadline).

The seam types, declared where they are consumed:

```go
// internal/browser
type Launcher interface {
    Open(url string) error
}

// internal/callback â€” the result of a completed redirect
type Result struct {
    Code string
}
```

`internal/oauth` takes its entropy as an `io.Reader` and its HTTP transport as
an `*http.Client`, both defaulted by its constructor when nil.

No package other than `cmd/oauth-login` writes to a process-global stream,
terminates the process, or reads the environment. Library packages return
errors; the composition root decides what that means for the user.

## Rejected.

- **Everything in `package main`.** Simplest to write, but the callback
  listener and the token exchange would then be reachable only through the
  whole flow, so their edge cases (IPv6 bind failure, stray paths, non-2xx
  bodies) could not be driven directly. It would also collapse the plan into
  one oversized phase.
- **A single `internal/login` package.** Keeps `cmd` thin but merges the two
  genuinely independent units, letting the listener and the protocol code
  reach into each other's state over time. The separation is what keeps the
  handshake testable without a socket and the listener testable without a
  provider.
- **Folding `internal/browser` into `internal/oauth`.** The launcher is the one
  seam that shells out to another process; keeping it alone makes it trivially
  fakeable and keeps `os/exec` out of the protocol package entirely.
- **Exported packages rather than `internal/`.** This repository ships a
  binary, not a library. Exporting the packages would create an API we would
  owe compatibility to, for consumers who are better served by executing the
  binary.
